1. Usage scenario: Git: Pulling a rebased branch

💯Score: 34    🔗Associated commands: [ git format-patch, git rebase, git branch, git checkout, git fetch, git push ]    🌐Details (opens new window)

Example:

▲ My recommendation (or, "what I would do if I were Mr Orange") is, start with git fetch.  Now I'll have this in my repo, which is what Mr Blond had after his rebase and just before he ran "git push -f".
M1  - M2 - M3
   \         \
    \         \
     A1 - A2   A1' - A2'

M1  ---- M2 ---- M3
   \               \
    \               \
     A1 - A2 - A3    A1' - A2'

▲ (with my local label pointing to A3 rather than A2; or A4 or A5, etc, depending on how many changes I have applied.)  Now all I have to do is rebase my A3 (and A4 if needed, etc) onto A2'.  One obvious direct way:
$ git branch -a
  master
* A
  remotes/origin/master
  remotes/origin/A
$ git branch new_A remotes/origin/A
$ git rebase -i new_A

$ git checkout -b new_A remotes/origin/A
$ git format-patch -k --stdout A3..A | git am -3 -k

M1  ---- M2 ---- M3
   \               \
    \               \
     A1 - A2 - A3    A1' - A2' - A3'

M1  ---- M2 ---- M3
   \               \
    \               \
     A1 - A2 - A3    A1' - A2' -- M
                 \_______________/

$ git branch -m A old_A
$ git branch -m new_A A

▲ and then, if old_A can be abandoned entirely:
$ git branch -D old_A

# 🔹 --directory

1. Usage scenario: merge a branch into a new sub directory on the master

💯Score: 2    🔗Associated commands: [ git apply, git checkout, git format-patch ]    🌐Details (opens new window)

Example:

$ git checkout master
$ git format-patch -o ../patches/ master..feature-branch
$ git am --directory=feature-subdir ../patches/*

   --directory=<root>
       Prepend <root> to all filenames. If a "-p" argument was also passed, it is applied before prepending the new root.

       For example, a patch that talks about updating a/git-gui.sh to b/git-gui.sh can be applied to the file in the working tree modules/git-gui/git-gui.sh by running git apply
       --directory=modules/git-gui.
